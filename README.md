# Cars Load Testing

Репозиторий для нагрузочного тестирования сервиса бронирования автомобилей [cars](https://github.com/marselchik73/cars).  
Содержит JMeter-скрипты, результаты тестов и HTML-отчёты.
---
## Структура репозитория

- `scripts/jmx/` – сценарии нагрузочных тестов (файлы `.jmx`) для разного количества виртуальных пользователей:
  - `load-testing10.jmx` - 10 пользователей
  - `load-testing100.jmx` - 100 пользователей
  - `load-testing500.jmx` - 500 пользователей
  - `load-testing1000.jmx` - 1000 пользователей
- `reports/` - папка с результатами выполненных тестов. Каждый запуск скрипта создаёт **новую подпапку с временной меткой** (например, `20260305_200610/`), внутри которой:
  - файлы `.jtl` - сырые данные
  - подпапки `report10/`, `report100/` и т.д. - готовые HTML-отчёты
- `run-all.sh` - скрипт для последовательного запуска всех тестов (10, 100, 500, 1000 пользователей)

---

## Как запустить тесты

### Требования
- Установленный **Apache JMeter** (в нашем случае путь `C:\JMeter\apache-jmeter-5.6.3\bin\jmeter.bat`)
- **Java**
- **Git Bash**

### Запуск одного теста (например, на 1000 пользователей)
Откройте Git Bash в корне репозитория и выполните:
```bash
"/c/JMeter/apache-jmeter-5.6.3/bin/jmeter.bat" \
  -n \
  -t "scripts/jmx/load-testing1000.jmx" \
  -l "reports/results1000.jtl" \
  -e -o "reports/report1000"

### Базовый URL

`https://testologia.ru`

### Эндпоинты

| Метод | Путь            | Описание                          | Параметры                                  |
|-------|-----------------|-----------------------------------|--------------------------------------------|
| GET   | `/cars-data`    | Получение списка автомобилей      | `filter` (опционально) – марка автомобиля |
| POST  | `/cars-order`   | Отправка заявки на бронирование   | JSON: `{ "car": "BMW M4 Competition", "name": "Иван", "phone": "81234567890" }` |
